Fairness Test Explanation

Now,I will tell you how to understand the files included "access_fairness" and use them to make a fairness test,which contains five aspects:gender,age,race,politics,and religion.

  • main.py
    This is the main program,integrating all algorithms to solve backend question banks using LLMs and scoring them accordingly.
  • setting.py
    This progarm includes APIs for the five LLMs we tested,with a unified format that is easy to call.
  • config.py
    This program writes a function named call_api(), which retrieves the LLM response by passing in the model name and prompt.
  • Toxicity.py
    This program primarily developed a toxicity score algorithm which is named detect_toxicity() for test responses using the martin-ha/toxic-comment-model model, with the function returning a toxicity score.
  • Politic.py
    This program includes an algorithm for detecting political stance biases, packaged as check_bias_in_model_output(), using a model of premsa/political-bias-prediction-allsides-BERT that returns a political stance, i.e., left-wing, right-wing, or neutral.
  • Sentiment.py
    This program includes an algorithm for testing textual positive emotion scores, packaged as an normalize_sentiment_score() using a sentiment-analysis model, with the function returning a positive positive emotional score. If two paragraphs of text are tested separately, subtracting the scores will reveal their emotional bias values.
  • Semantic.py
    The main content of this program is a semantic similarity algorithm packaged as compare_texts(), which uses an bert-base-chinese model and returns a similarity score. It performs similarity calculations on the two given paragraphs of text, allowing us to compare LLM's responses when facing different sensitive attributes, thereby determining fairness.
  • test.py
    This is a simple test program for evaluating the feasibility of optimization algorithms, allowing for easy archiving of different algorithms with the same functionality.
